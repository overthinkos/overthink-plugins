### 2026-07-01 — docs(go): RETRACT C13b (migrate as a bootstrap-phase auto-migrate plugin) as unsound

R5 claim-keyed sweep paired with the superproject's C13b retraction. RDD on the live code showed that making
`plugin-migrate` a bootstrap-phase plugin (serving `OpBootstrap` to auto-migrate a stale config's root bytes
in-memory at load) is architecturally unsound: the migrate chain is whole-project file-based + host-coupled and
needs host-prelifted inputs from a *completed* `LoadUnified`, but `runBootstrapPhase` fires *inside*
`LoadUnified` on root bytes only (a re-entry cycle; a partial pass would stamp HEAD on an incompletely-migrated
doc). So C13/M15 is COMPLETE as landed (migrate is `verb:migrate` over `OpRun`; the load gate keeps the explicit
`Run: charly migrate` reject), and the once-planned C13b is retracted.

- `/charly-internals:go` (go SKILL.md): the F9-bootstrap-phase paragraph no longer names migrate (or "M15") as
  a bootstrap plugin — it states that today only the no-op `candy/plugin-example-bootstrap` registers in the
  phase, and that migrate is `verb:migrate`/`OpRun` (the chain can't run on root bytes inside `LoadUnified`).
- The superproject sweep (its 2026-07 retraction entry) corrects the matching `.go` comments + CHANGELOG
  forward-claims; the 2026-06 plugins F9 entry above is annotated superseded.
